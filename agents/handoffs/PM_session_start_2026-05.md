## ДЕПЛОЙ И МЕРЖ

❌ **PM не участвует в деплое.**
✅ Деплой — зона ответственности QA.

**Workflow деплоя:**
1. PM формулирует ТЗ → передаёт в DEV
2. DEV реализует в feature branch → передаёт Preview URL в QA
3. QA тестирует → выдаёт APPROVED / REJECTED
4. **QA делает merge в main самостоятельно** (у QA deploy-токен)
5. PM получает уведомление от QA о выходе в production

PM не участвует в шаге 4. Решение о мерже принимает и исполняет QA.
