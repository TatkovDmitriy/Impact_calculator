Таблица содержит **три блока** данных, расположенных последовательно.

Доступные месяцы: `2025-12`, `2026-01`, `2026-02`, `2026-03`, `2026-04`

### Блок 2 — Основные метрики (Category × Segment × Month)

Строки: `Bathroom` → `Не Новосел` / `Новосел`, `Kitchen` → аналогично, `Storage` → аналогично.

Колонки (широкий формат pivot):

| Группа | Метрики внутри группы |
|---|---|
| `2025-12` (первый месяц) | 5.Средний чек \| 3.Конверсия% \| 2.Оплачено \| 1.Создано |
| `2026-01` … `2026-04` | 4.Выручка \| 5.Средний чек \| 3.Конверсия% \| 2.Оплачено \| 1.Создано |

> **Важно для парсера:** Выручка за месяц M расположена в колонке со заголовком следующего месяца `(M+1)/4.Выручка`. Для последнего месяца (апрель) выручка дублируется в trailing-колонке в конце строки. При парсинге надёжнее считать Выручку = Оплачено × АОВ.

**Структурные маркеры для парсера:**
- Строки-категории содержат ровно одно из: `"Bathroom"`, `"Kitchen"`, `"Storage"`
- Строки-сегменты содержат: `"Не Новосел"` или `"Новосел"`
- Строки месяцев в заголовке — паттерн `YYYY-MM`

### Блок 3 — Клиентские метрики (Segment × Month)

Строки: `Не Новосел`, `Новосел`

Колонки per month (в обратном порядке нумерации в заголовке):

| № | Метрика |

| 1 | Уникальных клиентов |

| 3 | Оплачено проектов (шт) |

**Kitchen AOV резко падает:** 211,919 (дек'25) → 166,299 (апр'26), −21.5% за 5 месяцев. Конверсия тоже: 64.9% → 44.1%. Это заметный тренд — возможно, сезонность или изменение программы.

**Bathroom Новосел конверсия падает:** 86.4% → 74.5%. При этом количество созданных сделок выросло: 2,382 → 5,331 (+124%). Программа привлекает больше клиентов, но конверсия снижается (более широкий, менее квалифицированный охват).

**Storage AOV Новосел:** 84,880 (дек) → 74,180 (апр). Ниже Kitchen, но выше не-новосёлов (44,538). Дисконт 7,418 — ниже капа 10k, программа работает органично.

**Не Новосел Kitchen конверсия обвалилась:** 33.7% → 18.2%. Потенциальная пища для анализа: конкуренция перетягивает конверсию или рынок сжимается.

## API-доступ

Используем **Google Sheets API v4** через `googleapis` npm-пакет. Аутентификация — Service Account (тот же что для Firebase Admin).

### Серверный клиент

`lib/sheets/client.ts`:

```typescript
import { google } from 'googleapis';

export function getSheetsClient() {
  const auth = new google.auth.JWT({
    email: process.env.GOOGLE_SHEETS_CLIENT_EMAIL,
    key: process.env.GOOGLE_SHEETS_PRIVATE_KEY?.replace(/\\n/g, '\n'),
    scopes: ['https://www.googleapis.com/auth/spreadsheets.readonly'],
  });
  return google.sheets({ version: 'v4', auth });
}

export async function fetchRange(range: string) {
  const sheets = getSheetsClient();
  const res = await sheets.spreadsheets.values.get({
    spreadsheetId: process.env.GOOGLE_SHEETS_SPREADSHEET_ID,
    range,
  });
  return res.data.values ?? [];
}
```

### Парсер (структурный, не по адресам ячеек)

`lib/sheets/parser.ts` — находит маркеры динамически:

1. Загрузить весь лист: `fetchRange('A1:ZZ500')`
2. Найти строку с заголовками месяцев (паттерн `YYYY-MM`) → определить позиции колонок
3. Найти строки-категории (`Bathroom` / `Kitchen` / `Storage`) по содержимому первой колонки
4. Для каждой категории — найти строки сегментов (`Не Новосел`, `Новосел`)
5. Извлечь метрики: Created, Paid, Conv, AOV, Revenue для каждого сегмента × месяц
6. Найти блок клиентских метрик (строки `Не Новосел` / `Новосел` в нижней части)

## Стратегия кеширования

1. **First read in a day** — fetch из Sheets API, сохранить в Firestore `metrics_cache/{YYYY-MM-DD}`
2. **Subsequent reads** — читаем из Firestore (быстрее + экономия квоты Sheets API)
3. **Force refresh** — кнопка в UI «Обновить метрики» — пере-фетч и перезапись `metrics_cache` за сегодня
4. **Сценарий сохраняется со снэпшотом** — в `scenarios/{id}.baselineSnapshot` копируется тот срез, что использовался при расчёте
