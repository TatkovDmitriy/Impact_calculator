# 07 — Спецификация дашборда

## Назначение

Дашборд — два режима, переключение через таб-навигацию:

1. **Overview** — точка входа: KPI из Sheets, тренд, recent scenarios, quick-launch
2. **Scenario Compare** — детализированный дашборд для сравнения сценариев одного или нескольких калькуляторов

## Анимации (Framer Motion)

### Overview-режим

| Элемент | Анимация | Триггер |
|---|---|---|
| KPI big numbers | Counter from 0 → final value за 1.2с, easeOut | Mount |
| KPI cards | Fade-in + slide-up, stagger 0.08с | Mount, sequential |
| Trend chart | Line draws left-to-right (1.5с) + area fade-in после линии | После загрузки данных |
| Scenario rows | Fade-in stagger | Mount |
| Calculator cards | Hover: scale 1.02 + shadow lift, accent border (#FDC300) | Hover |
| Page transitions | 0.25с fade + slight slide | Route change |

### Scenario Compare-режим

| Элемент | Анимация | Триггер |
|---|---|---|
| Delta table строки | Slide-in из левого края, stagger 0.06с | Mount / смена сценария |
| Delta числа | Counter анимация (включая отрицательные) | Каждый раз при изменении |
| Overlay chart — добавление сценария | Новая линия/бар вырастает из 0, остальные перерисовываются плавно | Toggle сценария |
| Overlay chart — удаление сценария | Линия fade-out 0.3с | Toggle off |
| Sensitivity chart — переключение input | Линии плавно морфируют к новым значениям, 0.6с | Смена оси X |
| Scenario pill (A / B / C) | Нажатие → border-highlight 0.2с + пересчёт | Click |
| Live-пересчёт в калькуляторе | Big numbers morph (layout animation Framer), 0.4с | Input change |

**Принцип Compare-режима:** пользователь должен **физически видеть**, как изменение одного параметра меняет итог. Это главная ценность для CPO-демо — «смотри, если мы поднимем долю Новоселов с 15% до 25%, net margin растёт вот так».

**Принцип:** анимация должна подчёркивать данные, а не отвлекать от них. Для CPO-питча (демо в Zoom) — каждое появление графика читается за 1.5с, числа — нарастают убедительно.

## Цвета графиков

Из бренд-палитры ЛП — без зелёного:

```css
--chart-1: #FDC300;   /* primary, accent — рост */
--chart-2: #2F3738;   /* baseline */
--chart-3: #8B8B8B;   /* secondary */
--chart-4: #C4C4C4;   /* tertiary */
--chart-5: #5A6166;   /* deep neutral */
```

Для негативных значений / падений — приглушённый красный `#B84A4A` (только когда явно нужно показать риск/убыток).

## Внутри калькулятора (страница `/calculators/[slug]`)

Двух-колоночный лейаут:

- **Слева (1/3):** форма ввода (shadcn `<Form>`), кнопка «Рассчитать», под ней — мини-блок «Используемый baseline» (показывает, какие метрики и из каких ячеек Sheets были взяты — для прозрачности)
- **Справа (2/3):** результат — big number + графики + таблица. До нажатия «Рассчитать» — пустое состояние с подсказкой. Любое изменение input → мгновенный live-пересчёт без нажатия кнопки (debounce 300ms).

Панель сценариев — под формой ввода:

```
┌─ Сценарии ─────────────────────────────────────────┐
│ [A: Базовый — 15%] [B: Агрессив — 25%] [+ Добавить]│
│                                                     │
│ Активный: B — Агрессивный рост                      │
│ label: [Агрессивный рост         ] [Сохранить]      │
│                                   [В Compare →]     │
└─────────────────────────────────────────────────────┘
```

**Workflow сценарного моделирования:**
1. Пользователь меняет inputs → live-пересчёт на правой панели
2. Нажимает «Сохранить» → сценарий записывается в Firestore с текущими inputs + result + baselineSnapshot
3. Переключается на другой сценарий (или «+ Добавить») → меняет inputs → сохраняет
4. «В Compare →» → переход на `/dashboard?compare=setId` или новый Scenario Set

Кнопки в шапке результата:
- **Сохранить сценарий** → пишет в Firestore `scenarios/`, добавляет в активный Set
- **В Compare →** → открывает Scenario Compare дашборд с текущим Set
- **Версия формулы:** v1.0.0 (для прозрачности)

## Адаптив

- Desktop (> 1024px): полный лейаут
- Tablet (768–1024px): KPI 2×2, остальное в одну колонку
- Mobile (< 768px): всё в одну колонку, KPI как scroll-карусель

Mobile — не приоритет, но не должен быть сломан.

## Empty states

- **Нет сценариев:** иллюстрация + кнопка «Запустить первый калькулятор»
- **Метрики не подгрузились:** баннер с reason + кнопка «Повторить»
- **Калькулятор требует метрику, которой нет в Sheets:** конкретное сообщение «Нет данных для X — заполни ячейку Y»
